Matter

A matter is an organized record for a client, containing all the information related to a specific case. A single client can have multiple matters.

  • new_case.php Create new matter page

    This is a page for crate new matter.

    The mdl_menu_left_new_case.php serves as a sidebar component, and it is one among many similar components that contribute to the overall structure.”

    Table for matter

    When we create new matter, our date saved into multiple table.

    Table NameSection Name & Description
    case_casesMatter Information
    members Client Information : client Information stored in this table.
    client_related_contactsRelated Contacts : If client has any related contact it stored in this table.

    Components Breakdown

    This page contains lot’s of component. let me breakdown.

    • Matter Information
      • case_file/case_info.php → This is matter information component.
        • crm/matter-status.php→ We can set type and workflow on there.
        • crm/matter-whiteboard.php
        • crm/track_matter.php → There is a track matter table.

    • Upcoming & Overdue Events

      crm/upcoming-and-overdue-events.php → When a consultation or any other future activity is created, it will be displayed in this component.

    • Client Information

      client-information.php → Client information is displayed in an input field in that section.

    • Email Settings

      email-settings.php → It displays multiple email options for selection.

      If the client has any optional emails and related contact emails, they are displayed in a tabular format under email settings. We can select an email for sending messages to multiple email addresses, either manually or automatically.

      • case_file/case_emails.php → The table component

        The emails are listed with a checkbox. If you select the checkbox and save, it will save the selected emails.

      The table email_selection stored the selected emails data.

    • Client Login Information

      crm/client-login-information.php This component is an input box that displays the client's password. We can change the client's login password from there.

    • Client Related Contacts

      crm/client-related-contact.php → This component enables us to add additional contacts related to the client. We can enter information about the individuals associated with the client.

      The table client_related_contacts contains those data/information.

    • Permission

      crm/permission.php → We can grant permission for a user to handle this matter through this component. When we permit a user, a notification will be sent to their dashboard, and the matter data will be displayed in the permitted user's personal dashboard section.


      When we change any permission, the updates are reflected in the
      case_cases table, which is the main table for matters.

    • Further Information

      We don’t know about this component.

    • Linked Matters with Client

      crm/linked-matters.php As you know, we can create multiple matters using the same email address. This component helps us identify which matters are connected to the same client. It displays the linked matters, and if we want to manually link a matter, we can do so as well.

      • case_file/previous_case.php → Automatically connected matters are displayed in this table. When we create a matter using a previous client's email ID, a popup will appear.


        When we click the
        "OK" button, it will open a new tab showing the previous matter. If the matter is different, we can return to our new matter in the left tab. After clicking the "Save" button, another popup will appear, such as:

        Now, if we click the "YES" button, the new matter will be created and automatically linked with the previous matter.

        In the case_cases table, the column "fkclientid" is used to associate matters with clients. When a new matter is created with a unique client email, a system-generated ID is assigned to fkclientid. However, if the client already exists in the system and the matter is different, the same fkclientid is used to maintain the association. On the backend, this ID is utilized to retrieve and display related matters in the Linked Matters with Client component, presenting them in a structured tabular format.

        You can see it : case_file/previous_case.php in this file.

      • case_file/case_connection.php → Manually connected matters shows in this table.

        module/mdl_connect_case.php→ Table

Email Automations

Before learning about Email Automation, it's important to understand Scheduled Tasks or Cron Jobs. These function like alarms, allowing us to set a PHP file on the server to run automatically at a specified time.

  • Example of Scheduled Task

    Step 1: Go to the Plesk Dashboard and select "Scheduled Tasks."

    Step 2: You can now add a new file or view previously scheduled files.

    Step 3: Set the file and schedule the execution time as shown in the image.

  • How to start automation

    When we create or open a matter, a popup will appear asking, "Do you want to run an automation right now?"



    Clicking the
    "Run Automation" button will open a popup where you can select a start date and choose an automation template. Once you click the "Start" button, the automation will begin running.

  • How to create automation template

    First, navigate to Settings, then click on Email Automation.


    Now, click on the
    "Create New" button.


    Look at this box—this is the first node. Click on
    "Action", then select "Create". A popup will then appear.

    This is the automation creation popup.
